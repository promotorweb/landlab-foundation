FLUXO CORRETO DE PRODUÇÃO DOS TEMPLATES
Projeto de origem (gerado por IA)

Cada landing page pode nascer em um projeto separado com React + Vite.

Regra obrigatória no vite.config.ts

O build precisa usar caminho relativo:

base: "./"

Sem isso, os assets quebram quando forem movidos para dentro de /templates/nome-do-template/.

Exemplo correto:
import tailwindcss from '@tailwindcss/vite';
import react from '@vitejs/plugin-react';
import path from 'path';
import { defineConfig, loadEnv } from 'vite';

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, '.', '');

  return {
    base: "./",
    plugins: [react(), tailwindcss()],
    define: {
      'process.env.GEMINI_API_KEY': JSON.stringify(env.GEMINI_API_KEY),
    },
    resolve: {
      alias: {
        '@': path.resolve(__dirname, '.'),
      },
    },
    server: {
      hmr: process.env.DISABLE_HMR !== 'true',
    },
  };
});
Build do template

No projeto original:

npm run build

Isso gera:

dist/
├── index.html
└── assets/
    ├── index-xxxxx.js
    └── index-xxxxx.css
ETAPA DE EXPORTAÇÃO PARA O LANDLAB

Depois do build, copiar o conteúdo de dist/ para:

FS-LandPages/templates/nome-do-template/

Exemplo:

FS-LandPages/templates/dentista/
├── index.html
└── assets/
    ├── index-xxxxx.js
    └── index-xxxxx.css
REGRAS IMPORTANTES PARA NÃO QUEBRAR TUDO
1. Nunca usar caminho absoluto para assets no build

Errado:

/assets/index-xxxxx.js

Certo:

./assets/index-xxxxx.js

Isso é resolvido com:

base: "./"
2. Nunca copiar só o index.html

Sempre copiar:

index.html
pasta assets

Se faltar a pasta assets, o template quebra.

3. Nunca mexer manualmente no HTML buildado sem necessidade

O index.html gerado pelo build normalmente já vem correto.

Só corrigir manualmente se houver erro real confirmado.

4. O repositório FS-LandPages não usa npm

Ele é um repositório estático.

Então:

não rodar npm install nele
não esperar package.json
não tratar ele como projeto Vite
não misturar com o projeto de origem gerado pela IA
5. Deploy da Vercel para FS-LandPages

Como o FS-LandPages é estático, a Vercel deve ser configurada assim:

Framework Preset: Other
Build Command: vazio
Install Command: vazio
Output Directory: .

Isso evita erro de npm install em projeto sem package.json.

FLUXO DE DEPLOY OFICIAL

Depois de copiar o template novo para FS-LandPages:

git add .
git commit -m "update template nome-do-template"
git push

Se o Git disser:

nothing to commit, working tree clean

significa que:

ou nada mudou
ou o arquivo foi alterado e depois voltou ao estado anterior

Se eu só quiser forçar deploy:

git commit --allow-empty -m "redeploy"
git push
PADRÃO DE DIAGNÓSTICO DE ERROS

Quando houver erro, você deve primeiro identificar qual destes casos é:

Caso A — erro no projeto original React/Vite

Exemplos:

erro de build
erro de TypeScript
erro no vite.config.ts
tela branca no projeto original
problemas com React

Nesse caso, pedir arquivos do projeto original, por exemplo:

vite.config.ts
package.json
src/main.tsx
src/App.tsx
src/index.css
Caso B — erro no LandLab estático

Exemplos:

template publicado não abre
CSS sumiu
tela branca depois do deploy
asset 404
JS buildado não carrega

Nesse caso, pedir arquivos do template exportado, por exemplo:

templates/dentista/index.html
estrutura da pasta templates/dentista/
nomes reais dos arquivos dentro de assets/
